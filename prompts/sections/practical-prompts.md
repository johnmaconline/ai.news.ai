# Practical Prompts

Prioritize practical prompts and prompt templates that software developers can apply immediately.

The requirements for this section:
1. Only include items that contain an explicit prompt asset or direct prompt discussion.
2. Valid examples: prompt text, system prompts, prompt templates, prompt packs, prompt repos, `agents.md`, `agent.md`, `claude.md`, cursor rules, copilot instructions, or prompt critiques for software development work.
3. Focus on code generation, test generation, code review, documentation, DevOps, CI/CD, debugging, refactoring, and repo automation prompts.
4. Exclude tool roundups, product comparisons, generic workflow advice, startup/funding news, opinion pieces, and general engineering articles unless they clearly publish or dissect a reusable prompt artifact.
5. If the source only says a tool is useful, or explains a workflow without showing the prompt asset, it does not belong here.

Writing guidance:
- Lead with the exact prompt asset being shared and what development task it improves.
- State whether the value is from the prompt itself, the prompt structure, or the surrounding prompt-operating instructions.
- Highlight expected output quality, implementation constraints, and when the prompt is likely to fail.
- Prefer concrete examples, before/after results, reproducible prompt structures, and source-linked prompt text.
- Keep claims factual and proportional to evidence.
